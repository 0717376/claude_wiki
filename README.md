# Personal Wiki Assistant

Персональная вики (как Notion) с AI-ассистентом по содержимому, на базе Claude CLI.

- **Слева** — дерево md-страниц (создание/переименование/перетаскивание).
- **По центру** — страница: просмотр и редактирование (raw markdown + предпросмотр), мгновенное автосохранение.
- **Справа** — чат с Claude в **единой сессии**. Спрашивайте по вики, просите создавать и править страницы. Ассистент ищет (Grep/Glob), читает и пишет файлы. Есть голосовой ввод.
- **Telegram-бот** (опционально) — тот же ассистент и та же сессия из Telegram, включая голосовые.
- Адаптивная вёрстка: на телефоне панели переключаются нижним таб-баром.

Один пользователь, вход по паролю. Содержимое — обычные `.md` файлы в `content/`.

## Архитектура

- `backend/` — FastAPI. Запускает `claude` CLI как subprocess с `--resume <session_id>`, стримит ответ по WebSocket. Файловое API для дерева/чтения/записи. Единая сессия хранится в `data/session.json`. Telegram-бот (long polling) и прокси распознавания речи — в том же процессе.
- `frontend/` — React + Vite, 3 панели. За nginx.
- Полный доступ инструментов ассистента (включая Bash) ограничен папкой `content/` (рабочая директория CLI). Веб и Telegram пишут в одну сессию, доступ к ней сериализуется.

## Запуск

```bash
cp .env.example .env
# отредактируйте WIKI_PASSWORD и WIKI_PORT
cp -r content.example content       # стартовое содержимое вики (ваше — остаётся локальным)
docker compose up --build -d
```

Откройте `http://<host>:<WIKI_PORT>` (по умолчанию `8842`), введите пароль.

Требуется авторизованный Claude CLI на хосте: каталог `~/.claude` пробрасывается в контейнер бэкенда. Если CLI ещё не авторизован — выполните `claude` на хосте и войдите.

> `content/` и `data/` не коммитятся (см. `.gitignore`) — ваши заметки и сессия остаются только у вас. В репозитории лежит демо в `content.example/`.

## Telegram-бот (опционально)

1. Создайте бота у **@BotFather** (`/newbot`), получите токен.
2. Впишите `TELEGRAM_BOT_TOKEN` в `.env`, перезапустите: `docker compose up -d backend`.
3. Напишите боту — он ответит вашим Telegram ID.
4. Впишите ID в `TELEGRAM_ALLOWED_IDS` (несколько — через запятую), снова `docker compose up -d backend`.

Бот понимает текст и голосовые, поддерживает `/clear`, `/compact`, `/help`. Контекст общий с веб-версией.

## Команды в чате

- `/clear` — очистить контекст (новая сессия).
- `/compact` — попросить CLI сжать историю (переслать как промпт).

## Переменные окружения (`.env`)

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `WIKI_PASSWORD` | пароль входа | — (обязательно) |
| `WIKI_PORT` | внешний порт фронтенда | `8842` |
| `CLAUDE_MODEL` | модель (`sonnet`/`opus`/`haiku`) | `sonnet` |
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather (пусто — бот выключен) | — |
| `TELEGRAM_ALLOWED_IDS` | разрешённые Telegram ID, через запятую | — |

Распознавание речи (голосовой ввод в вебе и Telegram) использует внешний ASR-сервис. Укажите свой эндпоинт в `ASR_UPSTREAM` (и при необходимости `ASR_MODEL`). Если переменная пуста — голосовой ввод просто отключён.

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `ASR_UPSTREAM` | URL ASR-сервиса (пусто — голос выключен) | — |
| `ASR_MODEL` | идентификатор модели ASR | `gigaam-rnnt` |

## Разработка

```bash
# backend
cd backend && uv run uvicorn main:app --reload --port 8000
#   нужны env: WIKI_PASSWORD, WIKI_DIR=../content, DATA_DIR=../data

# frontend
cd frontend && npm install && npm run dev   # http://localhost:5173 (проксирует на :8000)
```
